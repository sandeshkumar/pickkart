1 Overview

What We're Building

An AI-powered WhatsApp assistant that:

  • Auto-replies to customer queries 24/7
  • Checks product availability from PickKart's database
  • Helps customers place orders via WhatsApp
  • Sends order status updates
  • Handles FAQs (shipping, returns, payment)

3 VPS Server Setup

Step 3.1: Create a VPS

  1. Sign up at your chosen VPS provider (e.g., DigitalOcean)
  2. Create a new server:
    • OS: Ubuntu 22.04 LTS
    • RAM: 1GB minimum (2GB recommended)
    • Region: Bangalore / Mumbai (closest to India)
    • SSH Key: Add your SSH public key

Step 3.2: Connect to Your VPS

ssh root@YOUR_VPS_IP

Step 3.3: Initial Server Setup

# Update system packages
apt update && apt upgrade -y

# Create a non-root user
adduser pickkart
usermod -aG sudo pickkart

# Switch to new user
su - pickkart

Step 3.4: Install Node.js 20+

# Install Node.js via nvm (recommended)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Reload shell
source ~/.bashrc

# Install Node.js 20
nvm install 20
nvm use 20

# Verify installation
node --version   # Should show v20.x.x
npm --version    # Should show 10.x.x

Step 3.5: Install MySQL Client

sudo apt install -y mysql-client

Step 3.6: Test Database Connection

mysql -h YOUR_HOSTINGER_DB_HOST -u YOUR_DB_USER -p YOUR_DB_NAME \
  -e "SELECT COUNT(*) FROM products WHERE status='active';"
⚠️ Note: Your Hostinger database may only allow connections from specific IPs. You'll need to whitelist your VPS IP in Hostinger's Remote MySQL settings: hPanel → Databases → Remote MySQL → Add your VPS IP address.

4 Install OpenClaw

Step 4.1: Install OpenClaw CLI

# Install OpenClaw globally
curl -fsSL https://get.openclaw.ai | bash

# Or via npm
npm install -g @openclaw/cli

# Verify installation
openclaw --version

Step 4.2: Initialize OpenClaw

# Create workspace
mkdir -p ~/.openclaw/workspace
cd ~/.openclaw

# Initialize configuration
openclaw init

This creates the default directory structure:

~/.openclaw/
├── config.json5          # Main configuration
├── credentials/          # Channel credentials (auto-generated)
│   └── whatsapp/
├── workspace/
│   └── skills/           # Custom skills/plugins
│       └── pickkart/     # Our PickKart skill
└── logs/                 # Gateway logs

5 Connect WhatsApp

Step 5.1: Get a Dedicated Phone Number

⚠️ Important: Use a separate SIM card for the WhatsApp bot. Do NOT use your personal number. If WhatsApp flags the bot, your personal account stays safe.

Options:

  • Buy a new prepaid SIM (Jio/Airtel/Vi) — Rs 100-200
  • Use a virtual number service
  • Use a dual-SIM phone temporarily for setup

Step 5.2: Activate WhatsApp on the New Number

  1. Install WhatsApp on a phone with the new SIM
  2. Complete phone verification
  3. Set up a profile (name: "PickKart Support", photo: your logo)

Step 5.3: Link to OpenClaw

On your VPS, run:

openclaw channels login --channel whatsapp

This displays a QR code in your terminal. On the phone with WhatsApp:

  1. Open WhatsApp
  2. Go to Settings → Linked Devices → Link a Device
  3. Scan the QR code shown in your terminal

You should see:

[INFO] WhatsApp linked successfully!
[INFO] Account: +91XXXXXXXXXX
[INFO] Session stored at: ~/.openclaw/credentials/whatsapp/default/creds.json

Step 5.4: Verify Connection

openclaw channels status

# Expected output:
Channel    Account     Status
────────  ─────────  ───────
whatsapp   default     connected

6 Configure Access Control

Step 6.1: Edit Configuration

nano ~/.openclaw/config.json5

Step 6.2: WhatsApp Channel Configuration

{
  // Gateway settings
  gateway: {
    host: "127.0.0.1",
    port: 18789,
  },

  // Channel configurations
  channels: {
    whatsapp: {
      // === ACCESS CONTROL ===
      dmPolicy: "open",
      allowFrom: ["*"],

      // === GROUP SETTINGS ===
      groupPolicy: "disabled",

      // === MESSAGE SETTINGS ===
      sendReadReceipts: true,

      ackReaction: {
        emoji: "👀",
        direct: true,
        group: false,
      },

      chunking: {
        mode: "newline",
        limit: 4000,
      },

      // === MEDIA SETTINGS ===
      mediaMaxMb: 10,

      // === RATE LIMITING ===
      rateLimit: {
        maxMessages: 30,
        windowSeconds: 60,
      },
    },
  },
}

8 Database Query Tools

Step 8.1: Install MySQL Package

cd ~/.openclaw/workspace/skills/pickkart
npm init -y
npm install mysql2

Step 8.2: Database Connection Helper — tools/db.js

const mysql = require('mysql2/promise');

const DB_CONFIG = {
  host: 'YOUR_HOSTINGER_DB_HOST',      // e.g., srv1234.hstgr.io
  port: 3306,
  user: 'YOUR_DB_USERNAME',
  password: 'YOUR_DB_PASSWORD',
  database: 'u282948928_zedkart',
  waitForConnections: true,
  connectionLimit: 5,
};

let pool = null;

function getPool() {
  if (!pool) { pool = mysql.createPool(DB_CONFIG); }
  return pool;
}

async function query(sql, params = []) {
  const connection = await getPool().getConnection();
  try {
    const [rows] = await connection.execute(sql, params);
    return rows;
  } finally {
    connection.release();
  }
}

module.exports = { query };

Step 8.3: Product Search Tool — tools/check_product.js

const { query } = require('./db');

module.exports = {
  name: 'check_product',
  description: 'Search products by name, category, or brand',
  parameters: {
    type: 'object',
    properties: {
      search: { type: 'string', description: 'Search keyword' },
    },
    required: ['search'],
  },

  async execute({ search }) {
    const results = await query(
      `SELECT p.id, p.name, p.price, p.compare_at_price,
        p.stock_quantity, p.slug, c.name AS category, b.name AS brand
      FROM products p
      LEFT JOIN categories c ON p.category_id = c.id
      LEFT JOIN brands b ON p.brand_id = b.id
      WHERE p.status = 'active' AND p.stock_quantity > 0
        AND (p.name LIKE ? OR c.name LIKE ? OR b.name LIKE ?)
      ORDER BY p.is_featured DESC LIMIT 5`,
      [`%${search}%`, `%${search}%`, `%${search}%`]
    );

    return JSON.stringify({
      found: results.length > 0,
      products: results.map(p => ({
        id: p.id, name: p.name,
        price: `Rs ${Number(p.price).toLocaleString('en-IN')}`,
        inStock: p.stock_quantity > 0,
        url: `https://pickkart.co.in/products/${p.slug}`,
      })),
    });
  },
};

Step 8.4: Variants Tool — tools/check_variants.js

Queries product_variants table by product ID. Returns sizes, colors, prices, and stock for each variant.

Step 8.5: Order Status Tool — tools/check_order.js

Looks up orders by order number or phone number. Returns status, payment info, items, and delivery details.

Step 8.6: Category Listing Tool — tools/list_categories.js

Lists all active categories with product counts. Helps customers browse available product types.

Step 8.7: Order Placement Tool — tools/create_order.js

Creates a new order by inserting into orders and order_items tables. Handles stock validation, variant pricing, shipping calculation (free over Rs 499), and order number generation.

💡 Tip: Full code for all 5 tools is available in the OpenClaw-WhatsApp-Setup-Guide.md file included in your project. Copy the code blocks directly to your VPS.

11 Production Deployment

Step 11.1: Run as Background Service (systemd)

sudo nano /etc/systemd/system/openclaw.service
[Unit]
Description=OpenClaw WhatsApp Gateway
After=network.target

[Service]
Type=simple
User=pickkart
WorkingDirectory=/home/pickkart/.openclaw
ExecStart=/home/pickkart/.nvm/versions/node/v20.x.x/bin/openclaw gateway
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
⚠️ Note: Replace v20.x.x with your actual Node.js version. Find it with: which openclaw

Step 11.2: Enable and Start

# Enable and start the service
sudo systemctl daemon-reload
sudo systemctl enable openclaw
sudo systemctl start openclaw

# Check status
sudo systemctl status openclaw

# View logs
sudo journalctl -u openclaw -f

Step 11.3: Log Rotation

sudo nano /etc/logrotate.d/openclaw
/home/pickkart/.openclaw/logs/*.log {
    daily
    missingok
    rotate 14
    compress
    notifempty
    create 0640 pickkart pickkart
}

12 Monitoring & Maintenance

Daily Monitoring Commands

# Check if OpenClaw is running
sudo systemctl status openclaw

# View recent logs
sudo journalctl -u openclaw --since "1 hour ago"

# Check WhatsApp connection
openclaw channels status

# Check system resources
htop

Weekly Maintenance

# Update OpenClaw
npm update -g @openclaw/cli

# Restart service after update
sudo systemctl restart openclaw

# Check database connectivity
openclaw doctor

WhatsApp Session Management

WhatsApp linked device sessions can disconnect. If this happens:

# Check status
openclaw channels status

# Re-link if disconnected
openclaw channels login --channel whatsapp

# Restart gateway
sudo systemctl restart openclaw

Backup Credentials

cp -r ~/.openclaw/credentials ~/.openclaw/credentials.backup.$(date +%Y%m%d)

Rate Limiting Best Practices

🔐 To Avoid WhatsApp Flagging:
  • Don't send more than 1 message per second
  • Don't send bulk messages to people who haven't messaged first
  • Keep responses natural (not template-like)
  • Use a profile picture and proper business name
  • Start slowly (10-20 conversations/day) and gradually increase
